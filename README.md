# 🔍 Search Engine (C++)

## 📌 Описание проекта

Данный проект реализует упрощённую поисковую систему на языке **C++**.
Программа индексирует документы и позволяет выполнять поиск по запросам с вычислением релевантности.

Проект разработан в рамках финальной работы курса.

---

## ⚙️ Возможности

* 📄 Индексация текстовых документов
* 🔎 Поиск по ключевым словам
* 📊 Расчёт релевантности документов
* 📁 Работа с JSON-файлами (`config.json`, `requests.json`, `answers.json`)
* 🧪 Наличие тестового проекта

---

## 🧱 Структура проекта

```
search_engine/
│
├── src/                # Основная программа
│   ├── main.cpp
│   ├── SearchServer.cpp
│   ├── InvertedIndex.cpp
│   └── ConverterJSON.cpp
│
├── tests/              # Тесты
│   └── test.cpp
│
├── include/            # Заголовочные файлы и библиотеки
│   └── nlohmann/json.hpp
│
├── resources/          # Документы для индексации
│
├── config.json         # Конфигурация
├── requests.json       # Запросы
├── answers.json        # Результаты
│
├── CMakeLists.txt      # Главный CMake
└── README.md
```

---

## 🛠️ Требования

* C++17
* CMake ≥ 3.16
* Компилятор:

  * MinGW (Windows) / GCC / Clang

---

## 🚀 Сборка проекта

```bash
git clone https://github.com/sovmat31-png/search_engine_cpp.git
cd search_engine_cpp

mkdir build
cd build

cmake ..
cmake --build .
```

---

## ▶️ Запуск

### Основная программа

```bash
./src/search_engine.exe
```

или (если настроен вывод в build):

```bash
./search_engine.exe
```

---

### Тесты

```bash
./tests/search_engine_tests.exe
```

---

## 📂 Формат входных данных

### config.json

Содержит:

* список документов
* параметры поиска

---

### requests.json

Содержит поисковые запросы.

---

### answers.json

Формируется программой и содержит:

* результаты поиска
* релевантность документов

---

## 🧪 Тестирование

Тесты находятся в папке `tests/`.

Проект тестов собирается отдельно через CMake и не зависит от основной программы напрямую.

---

## 📌 Особенности реализации

* Используется **инвертированный индекс**
* Разделение на библиотеку и исполняемый файл
* Чёткая структура проекта (src / tests)
* Работа с JSON через библиотеку `nlohmann/json`
